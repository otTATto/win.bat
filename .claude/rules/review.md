# コードレビューに関するルールや考え方

- PR 全体を要約するコメントを除き、コード 1 箇所に対して 1 コメントでレビューする
- PR 全体を要約するコメントを除き、1 つのレビューコメントは以下のいずれかの prefix から始まる（`:` の後ろは prefix の説明文なのでコメントには不要）
  - [**WANT**]: できれば
  - [**MUST**]: 必須
  - [**IMO**]: 提案
  - [**FYI**]: 参考までに
  - [**NITS**]: 粗探し
  - [**Q**]: 確認
- PR 全体を要約するコメントを除き、レビューコメントにはコードの具体的な行数を指定し、可能であれば以下の形式で改善案を提案する

  ```suggestion

  ```
